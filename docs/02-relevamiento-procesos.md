# 02 – Relevamiento y Procesos Actuales  
Versión 1.0

---

# 1. Introducción

Este documento presenta el relevamiento actual del funcionamiento del negocio **AUMA**, abarcando los procesos comerciales, operativos y administrativos que forman parte de la actividad diaria. El objetivo es identificar cómo opera actualmente la marca, qué herramientas utiliza, qué problemas enfrenta y qué oportunidades de mejora surgen para la implementación del nuevo sistema e-commerce.

---

# 2. Descripción General del Negocio

AUMA es un emprendimiento artesanal dedicado a la creación y venta de productos aromáticos y de bienestar. Actualmente, el negocio funciona en modalidad manual y descentralizada, con fuerte dependencia de redes sociales para la comunicación, venta y gestión de pedidos.

La operatoria diaria incluye:

- producción manual de productos  
- interacción con clientes por mensajes directos  
- organización de pedidos en papel, chats o notas  
- coordinación personal de pagos y entregas  
- registro informal de ventas, inventario y costos  

El crecimiento de la marca requiere una mayor organización digital y un sistema integral que soporte la expansión.

---

# 3. Procesos Relevados

A continuación, se describen los procesos actuales identificados durante el análisis del negocio.

---

## 3.1. Proceso de Venta Actual

**Descripción:**  
El proceso de venta se realiza principalmente a través de mensajes en Instagram. El cliente consulta un producto y la vendedora responde manualmente.

**Pasos actuales:**  
1. El cliente envía un mensaje por Instagram.  
2. La vendedora responde con fotos, precios y opciones disponibles.  
3. El cliente elige un producto.  
4. La vendedora confirma stock manualmente.  
5. Se coordina forma de pago (transferencia en la mayoría de los casos).  
6. El cliente envía comprobante.  
7. Se coordina retiro o entrega.  
8. Se registra el pedido de forma informal (chat, cuaderno o notas).  

**Problemas detectados:**  
- Alto tiempo de respuesta.  
- Dependencia total de interacción manual.  
- Riesgo de errores en stock y coordinación.  
- Conversaciones desordenadas y difíciles de rastrear.  
- No existe historial centralizado de pedidos.

---

## 3.2. Proceso de Producción

**Descripción:**  
La producción se realiza artesanalmente en pequeños lotes, según demanda.

**Pasos:**  
1. Control del insumo base (cera, esencias, frascos).  
2. Producción según pedidos próximos o ferias.  
3. Etiquetado y preparación individual.  
4. Almacenamiento básico.

**Problemas detectados:**  
- Falta de registro digital de inventario.  
- Escasa previsión de insumos.  
- Dificultad para saber qué productos tienen mayor rotación.  

---

## 3.3. Proceso de Gestión de Stock

**Descripción:**  
El stock actual se maneja de forma manual.

**Pasos:**  
1. Registrar productos listos en notas/Excel.  
2. Actualizar stock después de vender (cuando se recuerda hacerlo).  
3. Verificar disponibilidad revisando físicamente.

**Problemas:**  
- No existe inventario en tiempo real.  
- Errores comunes al confirmar pedidos.  
- Falta de visibilidad para planificar producción.  

---

## 3.4. Proceso de Pagos

**Descripción:**  
Los pagos se coordinan por transferencia o efectivo.

**Flujo:**  
- Cliente pide alias.  
- Envía comprobante por chat.  
- Se valida manualmente.  

**Problemas:**  
- Validación lenta y manual.  
- Falta de integración automática con sistemas de pago.  
- No existen registros organizados.

---

## 3.5. Proceso de Entrega / Envío

**Actual:**  
- Coordinación uno a uno por chat.  
- Envíos por correo o mensajería local.  
- No hay seguimiento integrado.  

**Problemas:**  
- Mucha conversación manual.  
- Falta de tracking y estados claros para el cliente.

---

## 3.6. Proceso Contable / Registros

**Descripción:**  
Los registros se guardan de forma dispersa (notas, Excel, chats, fotos de comprobantes).

**Problemas:**  
- Información desordenada y difícil de rastrear.  
- No hay informes ni estadísticas.  
- Falta de control sobre ventas mensuales, costos y ganancias.

---

# 4. Herramientas Actuales

Actualmente, AUMA utiliza:

- Instagram como canal principal de ventas.  
- WhatsApp para coordinación de entregas.  
- Transferencias bancarias como método de pago.  
- Notas de celular / hoja de cálculo para inventario básico.  
- Agenda o cuaderno para registrar ventas.  

Estas herramientas cumplen funciones básicas, pero no escalan con el crecimiento del negocio.

---

# 5. Principales Problemas Detectados

- Falta de automatización en ventas y pagos.  
- Gestión manual del stock y riesgo de quiebre o sobreventa.  
- Ausencia de catálogo centralizado.  
- Pedidos desorganizados, sin historial unificado.  
- Tiempo operativo muy alto.  
- Experiencia del cliente dependiente de la respuesta manual.  
- Falta de panel administrativo y reportes.  

---

# 6. Necesidades Identificadas

A partir del relevamiento, se identifican las siguientes necesidades:

- Plataforma centralizada para visualizar productos.  
- Carrito y flujo de compra automatizados.  
- Sistema para registrar y consultar pedidos.  
- Integración con métodos de pago.  
- Panel administrador para controlar catálogo, stock y pedidos.  
- Base de datos con información estructurada.  
- Reducción del trabajo manual.  

---

# 7. Conclusión del Relevamiento

El análisis muestra que AUMA opera bajo procesos manuales y descentralizados, lo cual limita el crecimiento y la eficiencia del negocio. La creación de un sistema e-commerce completo permitirá organizar, automatizar y profesionalizar todos los aspectos clave: ventas, stock, pagos, producción, logística y comunicación con el cliente.

Este documento sirve como base para modelar los requerimientos del sistema y orientar el diseño del MVP y de las fases posteriores.
